# Dev Documentation

Документация для разработчиков по слоям приложения AnkeyIDM Employee Data Synchronization.

## 📚 Назначение

Эта документация помогает:
- Быстро вспомнить, что делает каждый слой
- Понять, как добавлять новую функциональность
- Разобраться в назначении классов и интерфейсов
- Найти примеры типичных задач

## 🗂️ Структура

Документация организована **по слоям приложения**:

```
docs/dev/
├── README.md                    # Этот файл
├── TEMPLATE.md                  # Шаблон для новых документов
│
├── layers/                      # Документация по слоям
│   ├── resolve-core.md         # Resolve: основная логика
│   ├── resolve-dsl.md          # Resolve: DSL механизмы
│   ├── transform-core.md       # Transform: основная логика
│   ├── transform-dsl.md        # Transform: DSL механизмы
│   ├── cache-core.md           # Cache: основная логика
│   ├── cache-dsl.md            # Cache: DSL механизмы
│   └── ...
│
└── guides/                      # Практические руководства
    ├── adding-new-op.md        # Как добавить новую операцию
    ├── creating-new-layer.md   # Как создать новый слой
    └── dsl-patterns.md         # Общие паттерны DSL
```

## 🎯 Как использовать

### Создать документацию для нового слоя

1. Скопируйте [TEMPLATE.md](./TEMPLATE.md)
2. Переименуйте в `название-слоя.md`
3. Заполните секции на основе реального кода
4. Сохраните в `docs/dev/layers/`

### Найти информацию о слое

Откройте соответствующий файл в `docs/dev/layers/`

### Узнать, как добавить функциональность

Смотрите секцию **"🛠️ Как расширять"** в документации слоя

## 📋 Чеклист для документации слоя

При заполнении шаблона обязательно включите:

- [ ] Назначение слоя (зачем он нужен)
- [ ] Расположение в кодовой базе
- [ ] Ключевые интерфейсы и их назначение
- [ ] Основные классы и сервисы
- [ ] Инструкции по добавлению новых компонентов
- [ ] Как минимум 1-2 практических примера
- [ ] Связи с другими слоями

## 🔄 Обновление документации

Документацию нужно обновлять при:
- Добавлении новых абстракций (интерфейсов, портов)
- Изменении способа расширения слоя
- Рефакторинге архитектуры слоя
- Обнаружении частых ошибок/вопросов

---

**Совет**: Документация должна быть **практичной**. Лучше краткое описание + рабочий пример, чем длинное объяснение без деталей.
