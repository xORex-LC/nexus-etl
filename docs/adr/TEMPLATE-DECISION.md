## 🎯 Решение

[Чёткое описание принятого решения]

**Пример**:
> Реализовать автоматическую топологическую сортировку датасетов на основе `dep_map` с использованием алгоритма Кана. Граф зависимостей строится при инициализации `CacheDependencyGraph`, порядок вычисляется один раз и кэшируется.

## 🏗️ Архитектурное решение

### Компоненты

**Новые классы/модули**:
- `CacheDependencyGraph` в `connector/domain/cache_core/cache_dependency_graph.py`
  - Метод `_topological_order()` - алгоритм Кана
  - Метод `refresh_order()` - публичный API

**Изменения в существующих компонентах**:
- `CacheRefreshPlanner` теперь использует `dependency_graph.refresh_order()`

### Интерфейсы

```python
class CacheDependencyGraph:
    def __init__(self, dep_map: dict[str, list[str]]): ...
    def refresh_order(self, scope: list[str] | None = None) -> list[str]: ...
    def topological_order(self) -> tuple[str, ...]: ...
```

### Поток данных

```
dep_map → [__init__] → [_topological_order (Kahn)] → cached _topo
                                  ↓
[refresh_order] → filter by scope → ordered list
```

| Файл | Изменение |
|------|-----------|
| `cache_dependency_graph.py` | Создан новый класс |
| `cache_refresh_planner.py` | Использует `dependency_graph.refresh_order()` |

### Ключевые методы

- [`CacheDependencyGraph._topological_order()`](../dev/layers/cache/cache-core.md#метод-_topological_order) - алгоритм Кана (24 строки, O(V+E))

### Инварианты

1. **Граф без циклов**: `_topological_order()` выбросит ValueError если цикл обнаружен
2. **Детерминированный порядок**: Для одного и того же графа порядок всегда одинаков
3. **Зависимости перед зависимыми**: Гарантируется топологической сортировкой

**Пример**:

**Известные ограничения**:
- Не поддерживает динамическое изменение графа в runtime (требуется пересоздание `CacheDependencyGraph`)
- Порядок между независимыми датасетами undefined (зависит от входного порядка)

**Риски**:
- ⚠️ Очень большое количество датасетов (>1000) может замедлить построение графа
  - **Митигация**: Граф строится один раз при инициализации, переиспользуется далее
- ⚠️ Некорректная конфигурация `dep_map` может привести к ValueError
  - **Митигация**: Fail-fast при инициализации, чёткое сообщение об ошибке
