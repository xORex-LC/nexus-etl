# CONFIG-PROBLEM-001: Перегруженный Settings-слой и неявные ошибки мерджа

> **Статус**: Закрыта (решена в [CONFIG-DEC-001](./CONFIG-DEC-001-modular-settings-and-slice-wiring.md))  
> **Дата фиксации**: 2026-02-12

## Суть проблемы

Конфигурация приложения была организована как один плоский объект с ручным merge, что приводило к:
1. Высокой связности между слоями.
2. Сложности сопровождения при добавлении новых параметров.
3. Рискам неявного поведения (в том числе на `0/False`).
4. Протеканию полного settings-объекта в команды и use-cases.

## Симптомы

1. Для добавления одного параметра требовались правки в нескольких несвязанных местах.
2. Источник итогового значения поля был неочевиден.
3. Ошибки конфигурации обрабатывались неоднородно.
4. Конфигурационные границы архитектуры не были защищены тестами.

## Влияние

1. Рост стоимости изменений и ревью.
2. Повышенный риск регрессий в wiring.
3. Усложнение онбординга и диагностики.

## Решение

Проблема закрыта решением [CONFIG-DEC-001](./CONFIG-DEC-001-modular-settings-and-slice-wiring.md):
1. Введена модульная модель `AppSettings` + slices.
2. Production path переведён на `loadAppSettings(...)`.
3. Настройки в команды/use-cases передаются только по профильным срезам.
4. Добавлены архитектурные guardrails.

## Связанные артефакты

1. `connector/config/app_settings.py`
2. `connector/delivery/cli/app.py`
3. `tests/architecture/config/test_settings_boundaries.py`
